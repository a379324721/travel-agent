# travel-agent 产品文档

> 版本:v2.0(2026-07-12)· 描述当前已上线能力
> 历史:v1.0 为规划文档,其路线图(P0–P4)已全部交付,见文末「版本历史」

## 1. 产品概述

**travel-agent** 是面向企业员工的差旅 AI 助手:通过自然语言对话完成差旅全流程——行程规划、费用预估、差标合规校验、出差审批、订票、一键报销,并基于公司制度文档回答政策问题。

一句话:**"下周从北京去上海出差" → 行程草稿 + 费用预估 + 差标校验 →(超标自动走审批)→ 对话内订票 → 行程结束一键报销。**

当前形态:自部署服务(FastAPI)+ Web 对话界面。供应商库存、OA 审批、员工身份为内置模拟系统,以适配层隔离,可替换为真实系统。

## 2. 目标用户与场景

| 用户 | 典型场景 |
|------|----------|
| 出差员工 | "我这个职级能坐什么舱位?" "帮我规划行程并订票" "一键报销" |
| 部门管理者 | 校验下属行程是否合规、金额是否需要事前审批 |
| 行政/财务 | 维护差旅制度文档,员工自助查询;报销单自动汇总订单 |

## 3. 功能说明

### 3.1 对话入口

- **Web 界面**(`/`):流式输出、工具调用状态卡片、多身份切换、会话管理、快捷指令
- **API**(`POST /api/v1/chat`):OpenAI Chat Completions 风格,支持 SSE 流式;`session_id` 维持多轮记忆,`user_id` 携带身份

### 3.2 身份识别

请求带 `user_id` 时自动从员工目录取身份(姓名/工号/职级/部门/电话)注入对话,全流程不再询问用户工号职级。内置模拟员工:

| user_id | 姓名 | 职级 | 部门 |
|---------|------|------|------|
| E001 | 王宁 | 经理 | 技术部 |
| E002 | 李雷 | 员工 | 销售部 |
| E003 | 韩梅 | 总监 | 产品部 |

### 3.3 行程规划与费用预估

结构化生成行程草稿(去程/返程航段 + 酒店)并预估总费用,自动附带差标校验结果。

### 3.4 差标合规校验

规则引擎逐项校验并输出警告:

- 舱位上限:员工经济舱 / 经理超经 / 总监·高管商务舱(国际长途)
- 酒店限额:一线城市(北上广深杭)800 元/晚,其他 500 元/晚
- 差补参考:按职级 300–800 元/天
- 提前预订:不足 7 天提示附加费风险
- 审批线:预估超 5000 元提示需事前审批
- 个人预算上限校验

### 3.5 制度问答(RAG)

差旅政策类问题自动检索制度文档原文作答(多路召回 + 去重排序),引用条款并标注「以公司制度为准」。制度文本在 `data/policies/*.md` 维护,`scripts/seed_policies.py` 一键入库——**改文档即改政策口径,无需改代码**。

### 3.6 出差审批(模拟 OA)

金额超审批线时对话内提交审批单,返回单号可查询。当前为模拟秒批;对接真实 OA 后改为异步审批回调。

### 3.7 订票(模拟库存,订单真实落库)

机票/酒店/火车搜索与下单。库存为演示数据源;**订单记录真实写入 PostgreSQL**(确认码、金额、员工、报销状态),供报销汇总与审计。

### 3.8 一键报销

一句"提交报销"即汇总该员工全部未报销订单,生成报销单(单号、总额、明细),订单自动标记已报销,防止重复提交。

### 3.9 会话记忆

按 `session_id` 持久化于 Redis(默认 7 天),客户端只传增量消息;超 token 预算自动摘要压缩,长对话不丢关键信息(城市/日期/金额/政策)。

## 4. 端到端流程示例

```
员工(E001 王宁):下周三去上海出差两天,帮我规划并看差标

助手:[提交报销] → RMB-XXXX,共 1 笔 1280 元,订单已标记报销
```

## 5. 能力边界与已知限制

- **模拟系统范围**:供应商库存(固定演示数据)、OA 审批(秒批)、员工目录(3 人)。均以适配层隔离,替换真实系统不动业务逻辑。
- 差标的**结构化规则**(舱位/限额/审批线数值)在代码中配置;RAG 覆盖的是制度问答口径。两处需同步维护。
- SSE 流式若被客户端中途断开,该轮对话不写入会话记忆。
- 意图识别为规则快车道(可选 LLM 慢车道未默认启用);未识别意图不影响功能(工具保持可用)。
- 检索重排(cross-encoder)未启用(依赖过重),当前为多路召回 + 分数排序。
- 部分 LLM 端点(如 DashScope 思考模式)不支持强制工具调用,由 `LLM_FORCE_TOOL_CHOICE=false` 退化为自动模式。
